TUN 模式 Steam 下载游戏，导致代理服务器流量消耗异常#

首先，在 TUN 模式下，所有的流量（包括 Steam 和其他应用的数据）都会被代理，包括下载更新、游戏数据等。这意味着即使你不通过 Steam 直接下载，其他流量也会被转发，增加流量消耗。

解决方案:

在 Clash 的 Profiles - Global Extend Script 在规则配置中添加以下内容：

  "DOMAIN,steamcdn-a.akamaihd.net,DIRECT",
  "DOMAIN-SUFFIX,cm.steampowered.com,DIRECT",
  "DOMAIN-SUFFIX,steamserver.net,DIRECT",
  "DOMAIN-SUFFIX,steamcontent.com,DIRECT"

配置完成后，你的规则配置应该类似于下图（部分）：

const CustomizationRule = [
    //"DOMAIN-SUFFIX,jetbrains.ai,US-手动选择",
    //"DOMAIN-SUFFIX,jetbrains.ai,ChatGPT",
    "DOMAIN-SUFFIX,jetbrains.ai,节点选择",
    "PROCESS-NAME,tailscaled,DIRECT",
    "PROCESS-NAME,tailscaled.exe,DIRECT",
    "DOMAIN-SUFFIX,mcdn.bilivideo.com,REJECT",
    "DOMAIN-SUFFIX,mcdn.bilivideo.cn,REJECT",
    "DOMAIN-SUFFIX,szbdyd.com,REJECT",
    "DOMAIN,steamcdn-a.akamaihd.net,DIRECT",
    "DOMAIN-SUFFIX,cm.steampowered.com,DIRECT",
    "DOMAIN-SUFFIX,steamserver.net,DIRECT",
    "DOMAIN-SUFFIX,steamcontent.com,DIRECT",
    // 配置 Chatgpt 使用美国（或其他没有被ban）的节点
    "DOMAIN-SUFFIX,chatgpt.com,ChatGPT"
];

解决方案 2:

如果还不能解决，则可以切换到 Clash 的 Connections 页面，同时开始下载游戏，找到流量使用最多的域名(Downloaded 降序排列)，如st.dl.eccdnx.com:80.

找到后按照之前的方法添加到规则中：

"DOMAIN,st.dl.eccdnx.com,DIRECT"

已经配置规则 gitlab 采用直连(DIRECT), 实际仍无法进行 git 操作#

已经配置：“DOMAIN-SUFFIX,gitlab.devops.[name].com,DIRECT”

问题根源在于 TUN 模式下的 DNS 劫持和 Fake IP 机制

简单来说, TUN 流程如下:

当 Git 客户端需要连接 gitlab.devops.[name].com 时，它首先会向 DNS 系统查询这个域名对应的 IP 地址, 而开启 TUN 模式后, 默认会使用 Fake IP

解决方案:

需要做的就是在配置文件中配置该域名不使用 Fake IP

在 clash 及其衍生工具中, 这项配置通常在以下字段中:

fake-ip-filter:
  - "*.lan"
  - "router.asus.com"
  # 在列表里加上下面这行
  - "gitlab.devops.[name].com"
