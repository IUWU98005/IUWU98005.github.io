Jenkins 基本介绍#

Jenkins 是自动发布部署软件的系统，提供 Web 页面供用户进行管理。在 Web 页面，用户可以进行编排自动发布任务、下载、编译代码和执行远程命令等操作。

Jenkins 插件（Plugins）#

Jenkins 支持多种插件，如Maven、npm等插件。可以在 Web 页面的Dashboard > Manage Jenkins > Plugins 下安装这些插件，以满足编译所需要的环境要求。

Shell 指令#

如果部分编译环境无法通过插件满足，则可以通过在 Build Steps 中输入类似以下的 Shell 命令执行编译:

sed -i "s/@@date@@/$(date '+%Y.%m.%d')/g" Code/Load.json
sh build.sh

Jenkins 安装#

在安装 Jenkin 前，需要确保设备上已经配置过 Java 环境，可以通过以下 Shell 命令检验:

java -version

如果没有显示 Java 的版本信息，则需要先手动安装 Java 并配置环境变量。

在 Docker 中下载并运行 Jenkins#

  1. 下载 jenkinsci/blueocean 镜像并使用以下 docker run 命令将其作为 Docker 中的容器运行:
docker run \
    -u root \
    --rm \
    -d \
    -p 8080:8080 \
    -p 50000:50000 \
    -v jenkins-data:/var/jenkins_home \ 
    -v /var/run/docker.sock:/var/run/docker.sock \
    jenkinsci/blueocean

Jenkins 发布任务#

Jenkins 发布任务既可以通过在 Web 页面点击触发，也可以通过远程 API 触发。

远程 API 触发#

可以在 Dashboard > Framework > [project-name] > Configuration > Build Triggers 下设置远程 API 触发。

如果代码存储在 git 上，则可以在 framework > [project-name] > Webhook Settings 下配置 Webhook，在 git 更新代码后，便会自动触发对应的 Jenkins 发布任务。

发布任务具体流程#

发布任务过程中，可以在 Dashboard > [project-name] > Console Output 下看到所有终端记录。

  1. 新建发布任务 Dashboard > New Item，并选择 Freestyle project（也可以选择 “copy from” 来创建和已有发布任务类似的任务）。

  2. 下载代码 Dashboard > [project-name] > Configuration-General > Source Code Management, 在 git 条目下填写对应的仓库地址。

    • Jenkin 默认只支持一个仓库的配置，如果需要配置多个仓库，则需要下载 Multiple SCMs 插件
    • Maven 仓库需要安装插件

    下载的代码和编译生成的文件，可以在 Dashboard > [project-name] > Workspace 下进行查看。

  3. 编译代码

    • Dashboard > [project-name] > Configuration > Build Environment 的 Build Steps 条目下编写 Shell 命令来执行编译。
    • 通过安装 Jenkins 插件来进行编译

  4. 远程部署程序

    • Dashboard > [project-name] > Configuration > Build Steps 下，通过 Shell 指令完成。通过 scp、ssh 完成文件发送、远程指令执行。
    • 通过 Jenkins 的 Publish Over SSH 插件完成。
      • 在 Dashboard > Manage Jenkins > System 下的 SSH Sever 添加远程服务器。
      • 在 Dashboard > [project-name] > Configuration > Build Steps 的 add post-build action > Send build artifacts over SSH 选择对应的远程服务器，配置需要发送的文件、远程执行的 shell 命令。

  5. 发布任务

    • Dashboard > Framework > Status 选择对应的任务，手动点击启动发布任务
    • 自动触发发布任务

      • Dashboard > [project-name] > Configuration > Build Triggers 配置对应的 API 地址（同时设置 API 用户名和 Token）

        用户 Token 需要在 Dashboard > root > Configure 的 API Token 中手动新建 Token。

      • 在 git 仓库中配置对应的 Webhook，以 gitlab 为例: Settings > Webhooks > Add new webhook 并勾选对应的触发事件。如 push events 会在仓库每次接收 push 时触发 Jenkins 发布任务。
