@classmethod 和 @staticmethod#

@classmethod 和 @staticmethod 是 Python 自带(builtin) 的两个装饰器

classmethod 和 staticmethod 都可以通过 object 调用和 class 调用， 也就是:

  1. 通过对象调用，A().f(1)
  2. 通过类调用，A.f(1)

@staticmethod#

staticmethod 就是普通函数放到 Class 的命名空间下，但不用传入 self 参数。

  1. 提供更好的封装，将功能绑定在类上，而不是仅仅是在类的对象上。

@classmethod#

对比普通类中的方法，classmethod 的第一个 arg 为 cls，而不是 self。

class Person1:

    ...

    @staticmethod
    def create(name, birth_year):
        curr_year = datatime.now().year
        age = curr_year - birth_year
        return Person1(name, age)

class Person2:

    ...

    @classmethod
    def create(cls, name, birth_year):
        curr_year = datatime.now().year
        age = curr_year - birth_year
        return cls(name, age)

class Child1(Person1):

    def __init__(self, name, age, department):
        super().__init__(name, age)

class Child2(Person2):

    def __init__(self, name, age, department):
        super().__init__(name, age)

以上两段代码的功能基本相同，第二段代码中的 cls 可以代表 Person2 自己。

两者的区别#

  1. 被 @classmethod 修饰的方法，返回 cls 对象，子类在调用时，返回的是子类自身（例子中为 Child2）；而 @staticmethod 修饰的方法，子类在调用时，返回的就是静态的类型（例子中为 Person1）

Python 线程池(ThreadPoolExecutor)#

ThreadPoolExecutor 与 Thread 的区别#

  1. Thread 在开始线程和结束线程时，会创建和销毁线程，这个过程会耗费大量的时间；ThreadPool 则省去了这两个步骤，直接使用原来的线程。

ThreadPoolExecutor 与 ProcessPoolExecutor#

ThreadPoolExecutor 适合处理 I/0 密集型操作，ProcessPoolExecutor 适合处理 CPU 密集型操作（需要大量计算）

基本用法#

ThreadPoolExecutor 为 Python 官方封装的 Threading 库，通过 from concurrent.futures import ThreadPoolExecutor 导入。

新建线程池#

from concurrent.futures import ThreadPoolExecutor
import time

def waste_time(n: int) -> None:
    print(f"{n} started...")
    time.sleep(n)
    print(f"{n} ended...")

t = ThreadPoolExecutor(max_workers=10)

for i in range(10):
    t.submit(waste_time, i)

print("end")
  • max_workers: 允许最多有几个线程运行，默认为 cpu 最大运行核心数*5（e.g. 8核cpu，此时 max_workers 默认为 40）
  • submit(func, args): args 为 func 的参数

这段代码可能先打印 end，之后再打印 “{n} started” 和 “{n} ended”，这是因为程序是非阻塞的。程序运行到 t.submit 时，程序继续运行 t.submit 的逻辑，剩余的代码则交给了另一个线程继续执行。

shutdown 的用法#

由于线程池是非阻塞的，如果要控制线程的执行顺序，可以使用 shutdown() 来控制线程的阻塞。

比如在上述代码 print(“end”) 上面一行加入 t.shutdown()，此时 end 就会等待所有线程都执行完成后才进行打印。

销毁线程#

可以将 shutdown 方法的参数 cancel_future 设置为 True，来进行线程的销毁。

with 关键词#

with ThreadPoolExecutor(max_workers=10) as t:
    for i in range(10):
        t.submit(waste_time, i)

other code...

使用 with 关键词有以下两个作用：

  1. 线程运行结束后，销毁线程
  2. 阻塞程序，等待所有线程执行完成后，才运行下面的代码

获取返回值#

通过新建列表 threads，并将线程添加到列表中，调用列表中元素的 result() 方法就可以获取到方法的返回值。

from concurrent.futures import ThreadPoolExecutor, as_completed
import time

def waste_time(n: int) -> int:
    print(f"{n} started...")
    time.sleep(n)
    print(f"{n} ended...")
    return n

threads = []

with ThreadPoolExecutor(max_workers=10) as t:
    for i in range(10):
    # t.submit(waste_time, i)
        threads.append(t.submit(waste_time, i))
    for thread in as_completed(threads):
        # 打印结果
        print(thread.result())

print("end")

任何在 as_completed() 被调用之前完成的 future 对象将会优先被生成。这段代码中会优先打印已经运行结束的线程的结果（可能不是按照1-10的顺序打印）。

超时(timeout)#

通过 timeout 参数设置超时，省去手动 try-catch 的过程

with ThreadPoolExecutor(max_workers=10) as t:
    for i in range(10):
    # t.submit(waste_time, i)
        threads.append(t.submit(waste_time, i))
    for thread in as_completed(threads, timeout=5):
        # 打印结果
        print(thread.result(timeout=5))

等待(wait)#

from concurrent.futures import ThreadPoolExecutor, as_completed, wait, ALL_COMPLETED, FIRST_COMPLETED
import time

def waste_time(n: int) -> int:
    print(f"{n} started...")
    time.sleep(n)
    print(f"{n} ended...")
    return n

threads = []

t = ThreadPoolExecutor(max_workers=10)

for i in range(10):
    threads.append(t.submit(waste_time, i))

# 当所有线程运行完毕后，才进行返回
wait(threads, timeout=10, return_when=ALL_COMPLETED)

print("end")

ALL_COMPLETED: 等待所有线程执行完毕后，运行下一行代码 FIRST_COMPLETED: 第一个线程执行完毕后，运行下一行代码

这段代码除了 print 打印的内容，还打印了 DoneAndNotDoneFutures 对象（包括已经完成的线程(done)和未完成的线程(not_done)）

这个特性适合做抢票的功能，即第一个人抢到票后，直接结束后面的线程：

from concurrent.futures import ThreadPoolExecutor, as_completed, wait, ALL_COMPLETED, FIRST_COMPLETED
import time

def waste_time(n: int) -> int:
    print(f"{n} started...")
    time.sleep(n)
    print(f"{n} ended...")
    return n

threads = []

t = ThreadPoolExecutor(max_workers=10)

for i in range(10):
    threads.append(t.submit(waste_time, i))

wait(threads, timeout=10, return_when=FIRST_COMPLETED)

t.shutdown()

print("end")
