@classmethod 和 @staticmethod#

@classmethod 和 @staticmethod 是 Python 自带(builtin) 的两个装饰器

classmethod 和 staticmethod 都可以通过 object 调用和 class 调用， 也就是:

  1. 通过对象调用，A().f(1)
  2. 通过类调用，A.f(1)

@staticmethod#

staticmethod 就是普通函数放到 Class 的命名空间下，但不用传入 self 参数。

  1. 提供更好的封装，将功能绑定在类上，而不是仅仅是在类的对象上。

@classmethod#

对比普通类中的方法，classmethod 的第一个 arg 为 cls，而不是 self。

class Person1:

    ...

    @staticmethod
    def create(name, birth_year):
        curr_year = datatime.now().year
        age = curr_year - birth_year
        return Person1(name, age)

class Person2:

    ...

    @classmethod
    def create(cls, name, birth_year):
        curr_year = datatime.now().year
        age = curr_year - birth_year
        return cls(name, age)

class Child1(Person1):

    def __init__(self, name, age, department):
        super().__init__(name, age)

class Child2(Person2):

    def __init__(self, name, age, department):
        super().__init__(name, age)

以上两段代码的功能基本相同，第二段代码中的 cls 可以代表 Person2 自己。

两者的区别#

  1. 被 @classmethod 修饰的方法，返回 cls 对象，子类在调用时，返回的是子类自身（例子中为 Child2）；而 @staticmethod 修饰的方法，子类在调用时，返回的就是静态的类型（例子中为 Person1）
