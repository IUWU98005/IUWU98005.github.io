Python `__repr__` 和 `__str__`

February 2, 2026 · 1 min
class Person(Object):
	def __init__(self, name: str):
		self.name = name

p = Person("Alexander")

上面这段代码，在终端打印时，打印的结果是对象的内存地址，类似以下输出：

>>> print(p)
<__main__.Person object at 0x10f29b940>

>>> p
<__main__.Person object at 0x10f29b940>

>>> p.__str__()
<__main__.Person object at 0x10f29b940>

>>> p.__repr__()
<__main__.Person object at 0x10f29b940>
  1. 重写 __str__ 方法
class Person(Object):
	...
	def __str__(self):
		return f"__str__ 方法, name: {self.name}"

p = Person("Alexander")

此时再次在终端打印，此时的输出结果变成以下情况：

>>> print(p)
__str__ 方法, name: Alexander

>>> str(p)
__str__ 方法, name: Alexander

>>> f"{p}"
__str__ 方法, name: Alexander

>>> p.__str__()
__str__ 方法, name: Alexander

>>> p
<demo.Person object at 0x10e0e3588>
  1. 重写 __repr__ 方法
class Person(Object):
	...
	def __repr__(self):
		return f"__repr__ 方法, name: {self.name}"

p = Person("Alexander")

直接输出对象：

>>> p
__repr__ 方法, name: Alexander

如果只重写了 __repr__ 方法，则所有调用 __str__ 的时机都会调用 __repr__ 方法
